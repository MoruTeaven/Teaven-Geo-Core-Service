## 修复远程 D1 数据库缺少表的问题
- **问题**: 远程 Worker 报错 `D1_ERROR: no such table: locations`，远程 D1 只有空库，从未执行过 schema 迁移
- **解决**: 
  1. 使用 Cloudflare API Token + Account ID 认证
  2. 执行 `npx wrangler d1 execute teaven-geo-db --remote --file=db/migrations/001_init.sql` 建表（14 条 SQL，4 张表）
  3. seed.sql 含 `BEGIN TRANSACTION` 语句 D1 不支持，写脚本过滤后生成 `seed_d1.sql`
  4. 执行 `npx wrangler d1 execute teaven-geo-db --remote --file=data/processed/seed_d1.sql` 导入 100 万行数据
- **结果**: 数据库 99.60MB，locations/location_names/path_cache/location_search 四表就绪
- **启示**: 部署前务必在远程 D1 也执行 schema 迁移，不能只做本地

## 添加 zh-Hant 繁体中文支持
- **方案**: 数据库仍用 zh 数据，Worker 运行时做简→繁字符级转换
- **新增文件**: `src/utils/s2t.ts` — 约 2500 组简繁字符映射表
- **修改文件**:
  - `src/utils/normalize.ts`: 新增 `isTraditionalChinese()` 判断和 `resolveLangPriority()` 中 zh-Hant → zh 映射
  - `src/services/geo.ts`: `getChildren`/`getLocation`/`getAncestors` 三个方法适配 zh-Hant 转换
- **支持的语言代码**: zh-Hant, zh-TW, zh-HK, zh-MO
- **注意**: 移除了一对多映射中不适用于地名的条目(如 `蒙:矇`，地名"内蒙古"应保持"蒙古"不变)

## 修复 /geo/children 无 parent_id 时返回 admin3 而非国家的Bug
- **根因**: `queryChildren` 中用 `parent_id IS NULL` 判断顶级，但 13W+ admin3 的 `parent_id` 也是 NULL（hierarchy.txt 未覆盖全部关系）
- **修复**: `queries.ts` 中将条件从 `l.parent_id IS NULL` 改为 `l.level = 'country'`
