## 2026-06-01

### Bug Fix: `t3 is not a function` 500 错误
- **根因**: `src/index.ts` 中从 `itty-router` v5 导入了 `error` 函数，但 v5 不再导出该函数。minify 后 `error` 变成了 `t3`，运行时为 `undefined`，调用触发 `t3 is not a function`。
- **修复**: 移除 `error` 导入，将所有 `error(status, msg)` 替换为 `json({ error: msg }, status)`（`json` 在 v5 中可用）。
- **影响范围**: `src/index.ts` — 所有错误响应路径。

### Bug Fix: GET / 返回 500 (cors 用法错误)
- **根因**: `cors()` 返回 `{ corsify, preflight }` 对象，直接放入 `before` 数组导致 `AutoRouter` 将其当作函数调用时抛出 `TypeError: object is not a function`，被默认 catch 捕获后返回 500。不仅影响 `/`，所有路由都有此问题。
- **修复**: 解构 `cors()`，将 `preflight` 放入 `before`，`corsify` 放入 `finally`。
- **影响范围**: `src/index.ts` 第 34-36 行。

### Bug Fix: json(data, statusCode) 报错 `constructor parameter 2 is not of type 'InitializerDict'`
- **根因**: itty-router v5 的 `json(data, init)` 第二个参数必须是 `ResponseInit` 对象（如 `{ status: 500 }`），不能直接传数字。源码 `new Response(body, init.url ? undefined : init)` 中，数字 `500.url` 为 `undefined`，导致直接传数字给 Response 构造函数。
- **修复**: 新增本地包装函数 `respond(data, status = 200)` → `json(data, { status })`，全文件统一替换 `json(` → `respond(`。
- **影响范围**: `src/index.ts` — 全部 14 处 json() 调用。
